üîê Apa Itu Private Method?

Private method adalah fungsi (method) didalam class yang tidak bisa diakses dari luar class. Sama seperti private field, private method juga ditandai dengan # di depan nama method -nya

Tujuan

Menyembunyikan logika internal dari class agar tidak bisa dipanggil sembarangan.

Menjaga enkapsulasi (encapsulation) ‚Äî prinsip OOP agar data dan proses di dalam class tetap aman.

Contoh


class User {
	#cekPassword(password) {
		// Ini adalah private method
		return password === "rahasia";
	}
	
	login(inputPassword) {
		if (this.#cekPassword(inputPassword)) {
		console.log("Login berhasil!");
		} else {
		console.log("Password salah.");
		}
	}
	}
	
	const user1 = new User();
	
	user1.login("rahasia");     // ‚úÖ Output: Login berhasil!
	user1.login("salah");       // ‚úÖ Output: Password salah.
	
	// user1.#cekPassword("rahasia"); // ‚ùå ERROR! Tidak bisa akses private method

üìå Analogi untuk Pemula:

Bayangkan kamu punya mesin kopi otomatis. Di dalam mesin itu, ada tombol ‚Äúbuat kopi‚Äù, dan kamu cukup pencet satu tombol.

Tapi, di dalam mesin, ada banyak proses kecil: memanaskan air, menggiling biji kopi, mencampur air dan bubuk, dll. Nah, semua proses kecil ini adalah private method ‚Äî kamu tidak bisa dan tidak perlu tahu detailnya, cukup pencet 1 tombol saja (public method login()).

üß† Kapan Harus Menggunakan Private Method?

Kamu punya kode logika internal yang tidak seharusnya diakses langsung oleh pemakai class.

Kamu ingin memisahkan tugas-tugas internal dalam class agar lebih rapi dan modular.

Kamu ingin menyembunyikan detail implementasi dan hanya menyediakan interface yang jelas.
