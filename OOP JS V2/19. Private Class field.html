Private class field

Versi chatgpt

Apa itu Private Class Field?

Field privat di JavaScript adalah properti yang hanya dapat diakses di dalam kelas tempat properti tersebut dideklarasikan. Tidak seperti public class field yang bisa diakses dari luar kelas menggunakan this, private class field hanya bisa diakses melalui metode atau konstruktor di dalam kelas.

Untuk mendeklarasikan private field, kita menggunakan simbol # di depan nama field-nya. Ini adalah cara untuk memberi tahu JavaScript bahwa properti tersebut bersifat privat.

Kenapa Private Class Field Penting?

Private class field sangat berguna untuk menyembunyikan data di dalam kelas yang tidak seharusnya diubah atau diakses langsung dari luar kelas. Ini adalah prinsip encapsulation (pengkapsulan), yang merupakan konsep penting dalam pemrograman berorientasi objek. Dengan cara ini, kita bisa melindungi data internal kelas agar tidak mudah diubah secara sembarangan oleh kode luar.

Contoh


class Mobil {
	#warna;  // Private field

	constructor(merek, warna) {
		this.merek = merek;  // Public field
		this.#warna = warna; // Menyimpan warna mobil di private field
	}

	tampilkanWarna() {
		console.log(`Warna mobil adalah: ${this.#warna}`);  // Akses field privat dengan metode dalam kelas
	}

	// Method untuk mengubah warna mobil
	setWarnaBaru(warna) {
		this.#warna = warna;  // Mengubah warna mobil menggunakan metode
	}
}

const mobil1 = new Mobil("Toyota", "Merah");

mobil1.tampilkanWarna(); // Output: Warna mobil adalah: Merah

// mencoba mengakses #warna langsung dari luar kelas
// mobil1.#warna = "Biru"; // Error: Cannot access private field #warna

mobil1.setWarnaBaru("Biru"); // Mengubah warna mobil melalui metode kelas
mobil1.tampilkanWarna(); // Output: Warna mobil adalah: Biru

Penjelasan:

#warna adalah private field. Field ini hanya dapat diakses di dalam kelas Mobil.

Di constructor, kita menginisialisasi #warna dengan nilai yang diteruskan sebagai parameter.

Metode tampilkanWarna() bisa mengakses #warna untuk menampilkan warna mobil.

Kita tidak bisa mengakses #warna langsung dari luar kelas menggunakan mobil1.#warna (akan menghasilkan error).

Untuk mengubah nilai #warna, kita bisa menggunakan metode setWarnaBaru() yang disediakan di dalam kelas.

Analogi untuk Pemula:

Bayangkan kamu memiliki sebuah kotak (kelas), dan di dalam kotak tersebut ada beberapa barang (field). Ada beberapa barang yang kamu ingin agar hanya kamu yang bisa mengakses dan merubahnya. Barang-barang ini adalah barang-barang pribadi, dan hanya bisa kamu pegang, bukan orang lain di luar kotak.

Di dalam kotak, kamu punya metode (cara) untuk mengubah barang tersebut (misalnya, jika kamu ingin mengganti warna mobil). Itu berarti kamu bisa mengontrol siapa yang boleh melihat atau mengubah barang tersebut dengan memberikan akses hanya melalui metode yang kamu buat di dalam kotak.

Namun, barang pribadi ini tidak bisa dilihat atau diubah langsung oleh orang lain dari luar kotak. Ini adalah private field dalam istilah programming.

Kapan Menggunakan Private Fields?

Saat kamu ingin melindungi data internal kelas agar tidak bisa diubah atau diakses sembarangan dari luar kelas.

Saat kamu ingin menjaga integritas data dengan memastikan bahwa perubahan hanya dilakukan melalui metode yang kamu kontrol.

Kesimpulan:

Private fields adalah fitur untuk membatasi akses data di dalam kelas.

Menggunakan # untuk mendeklarasikan field privat.

Field privat hanya bisa diakses atau diubah melalui metode di dalam kelas, bukan dari luar kelas.

Versi PZN

Secara default saat kita menambahkan field, maka field tersebut bisa di akses dari mana saja

Jika kita ingin membuat field bersifat private (hanya bisa diakses dari class block), kita bisa menggunakan tanda # sebelum nama field nya

Ini dinamakan prvate class field, dan hanya bisa diakses di class nya saja


class Counter {
	#counter = 0;

	increment () {
		this.#counter++;
	}

	decrement() {
		this.#counter--;
	}

	get() {
		return this.#counter;
	}
}
