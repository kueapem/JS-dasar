Static class field

Versi Chatgpt

üîç Apa itu Static Class Field?

adlaha kata kunci yang digunakan untuk membuat properti atau method yang dimiliki oleh class itu sendiri, bukan oleh instance-nya.

Jadi, kalau objek biasa menyimpan data di this, maka static menyimpan data langsung di class-nya.

üéì Analogi Sederhana

Bayangkan:

Class = Pabrik

Instance = Barang hasil pabrik

Static = Data yang hanya dimiliki pabrik, bukan oleh barangnya

Contoh

Pabrik Mobil (class) punya jumlah total produksi (static field: totalMobil)

Tapi tiap mobil yang dibuat (instance) tidak tahu-menahu tentang berapa banyak mobil yang sudah dibuat.

‚úÖ Contoh Kode: Static Field dan Static Method


class Mobil {
	static totalMobil = 0; // static field
	
	constructor(nama) {
		this.nama = nama;
		Mobil.totalMobil++; // akses static field lewat class, bukan this
	}
	
	static infoProduksi() {
		console.log(`Total mobil yang diproduksi: ${Mobil.totalMobil}`);
	}
	
	// method biasa
	info() {
		console.log(`Ini mobil ${this.nama}`);
	}
}
	
const m1 = new Mobil("Avanza");
const m2 = new Mobil("Innova");

m1.info(); // Ini mobil Avanza
m2.info(); // Ini mobil Innova

Mobil.infoProduksi(); // Total mobil yang diproduksi: 2

console.log(m1.totalMobil); // undefined! (karena ini milik class, bukan instance)
Element Penjelasan
static totalMobil = 0 Field milik class, bukan objek
Mobil.totalMobil++ Akses dan ubah static field
Mobil.infoProduksi() Static method hanya bisa dipanggil lewat nama class
m1.totalMobil ‚ùå undefined, karena objek m1 tidak punya properti totalMobil

üß™ Catatan Penting

Static field/method tidak bisa diakses lewat this di dalam instance method

Static method juga tidak bisa mengakses this.nama (property dari instance) ‚Äî karena tidak tahu instance mana yang dimaksud.

üß† Kapan Gunakan Static Field?

Untuk menyimpan data global milik class, seperti jumlah instance, pengaturan default, atau cache.

Untuk membuat utilitas atau fungsi bantu dalam class (tanpa harus membuat objeknya).

Static class field

Jika kita tambahkan static dala class field, secara otomatis field tersebut bukan lagi miliki instance object, melainkan milik class nya itu sendiri.

Biasanya static digunakan jika kita ingin membuat utility field atau function

Cara mengakses static class field pun tidak lagi lewat obeject, melainkan lewat class nya

Static class field bisa diartikan sifaftnya global, tidak peduli di akses dimana atau siapa yang mengakses, hasilnya akan sama

Contoh


class Configuration {
	static name = "Belajar Js Dasar";
	static version = 1.0;
	static author = "Kenny";
}

// cara aksesnya
console.info(Configuration.name);
console.info(Configuration.version);
console.info(Configuration.author);

// merubah nilainya
Configuration.name = "Rui";
