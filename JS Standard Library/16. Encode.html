Encode

Saat kita menuliskan Url, kadang kita ingin menambahkan informasi tambahan seperti query parameter misalnya

Url sendiri sudah memiliki standar encoding penulisan URL

Standar encoding ini dilakukan agar penulisan URL aman ketika diterima oleh server

Aman disini dalam artian informasi URL tidak berubah

Contoh paling sederhana, misal walaupun kita bisa mengirim spasi dalam url, tapi disarankan untuk di encode agar nilai spasi tidak benar-benar terlihat seperti spasi pada URL nya

Conoth masalah jika tidak menggunakan Encode

Misal ada query parameter dengan nama data, lalu kita ingin mengirimkan nilai kesana

Misal nilainya adalah &eko=eko&, alhasil URL nya akan seperti berikut

https://contoh.com?data=&eko=eko&

URL diatas seperinya tidak ada masalah, tapi sebenarnya ketika diterima oleh server, parameter data bernilai kosong, kenapa? karena & dianggap pemisah antar parameter

Artinya tanda seperti & dan lain lainnyta perlu di encode, agar tidak terjadi kesalahan data saat kita kirim

encodeURI(value) || Untuk encode value, namun karakter yang dipesan di URI tidak akan diubah ;/?:@&=+$,# encodeURIComponent(value) || Melakukan encode value semua karakter decodeURI(encoded) || Melakukan decode value hasil encodeURI() decodeURIComponent(encoded) || Melakukan decode value hasil encodeURIComponent()
