Default dan Named Export

Dalam satu module, kita bisa menggabungkan export default dan export named

Yang tidak boleh adalah, dalam satu module, membuat lebih dari satu defautl

Cara import default dan named secara bersamaan bisa menggunakan perintah berikut:


import defaultAlias, {named1, named2} from "./app.js";
import {default as defaultAlias}, {named1, named2} from "./app.js";

Rekomendasi

Bedakan antara default dan named, karena jika digabungkan kadang akan membingungkan ketika melakukan import

Jika kita menggunakan default, kita tidak perlu menuliskan nama variabel yang sama dengan nama exportnya

Jika kita menggunakan named, kita harus menuliskan nama variabel yang sama dengan nama exportnya

Sebisa mungkin selalu gunakan export named dibandingkan export defautl
